J-Star

国際化

J-Starでは国際化に対応することが可能です。 つまり、ブラウザの言語を変更すると言語に応じた画面に変更されます。 （ただし、この機能はローカルモードでは正しく機能しません）

国際化
概要
$$.setupオブジェクトにメッセージファイルを指定するとJ-Starがロケールを補完して、 メッセージファイルを読み込みます。 読み込んだメッセージは、メッセージオブジェクトとしてinit関数の第２引数として渡されます。
つまり、メッセージオブジェクトがブラウザのロケールによって取る値が異なることになります。 そしてこのメッセージオブジェクトを $$.fill 関数などに渡すことによって画面内の文言を書き換える事ができます。


国際化機能をOFFにするには jquery.suziQ.settings.js で　 
jQuery.sq.settings.locale_path=null;
として下さい。
メッセージファイル
国際化されたメッセージファイルを読み込むには、setupオブジェクトのmessagesで指定することができます。 詳細な使い方に関しては、setupオブジェクトの説明を参照してください。

メッセージファイル内の記述方法
メッセージファイルは以下の記述方法に従って下さい。 
  jQuery.sq.message({
    "aaa": "XXXXX",
    "bbb": "YYYYY"
  });
jQuery.sq.message関数の呼び出しで引数にオブジェクトをいれてください。 このように記述すると、メッセージオブジェクトから以下のようにデータを取得することができます。
alert(messages["aaa"]);
こうすると XXXXX と表示されます。

ロケールの決定方法
J-Starではロケールの取得方法は、サーバに問い合わせて決定するようになります。 どのような処理であるかは、J-Star付属のJavaのサンプルコード（LangServlet.java）を参照してください。 この実装では、HTTPヘッダーのAccept-Languageを利用するようになっています。

このサービスはサーバ側に配置する必要があります。 このサービスにアクセスするURLは、jquery.suziQ.settings.js で　 
jQuery.sq.settings.locale_path='/locale';
のように指定することができます。

ロケールをサーバに問い合わせるため、ローカルモードでは国際化が正確に機能しなくなっています
