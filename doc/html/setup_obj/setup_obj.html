setupオブジェクト

setupオブジェクトはJ-Starの最も基本的で重要なオブジェクトです。 このsetupオブジェクトのinit関数を定義することにより、J-Starが機能するようになります。

setupオブジェクトに設定できる値は以下の通りです。

init
この値は必須です。
画面に対する挙動のすべてをこのinit関数内で行ってください。

init関数の引数は２つです。最初の引数は前画面から引き継がれるデータです。より正確には$$.ajax.to関数（画面遷移を起こす関数）で渡されるデータになります。 ２つ目の引数は、メッセージオブジェクトになります。メッセージオブジェクトはsetupオブジェクトのmessagesに指定されたメッセージを格納したオブジェクトになります。
messages
messagesには、読み込みたいメッセージファイルを配列で指定してください。
この値は必須ではありません。

メッセージファイルの指定は、やや特別です。これは各ブラウザのロケールにあったファイルが読み込まれるようになるためです。
例えば以下のような指定の場合 
$$.setup ={
  messages: ["../js/aaa","bbb"],
  init : function(data,messages){}
};
このブラウザのロケールが「ja」だとすると、実際に読み込まれるファイルは
../js/aaa_ja.js と bbb_ja.js の２つになります。
つまり、messagesで指定された値に対して「_ja.js」が付加されたファイルが読み込まれます。 もちろん ja 部分はロケールによって変わります。

また、一度読まれたファイルは２回読み込まれることはありません。


国際化機能をOFFにした場合、「_ja」の部分が省かれます。先ほどの例でいえば
../js/aaa/js と bbb.js の２ファイルになります。
dummy_data
この値は、ローカルモードでのみ有効になります。前画面からのデータをMock化することができます。
この値は必須ではありません。

この値はinit関数の第１引数に渡されますが、前画面が存在しない場合、つまりローカルモードで一番最初に開いたHTML内でのみ使用されます。
