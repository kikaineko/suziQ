J-Star

J-Starとは

サーバと画面を完全に分離することによりAgileな開発を可能にするJavascriptフレームワークです。

J2EE 標準の Web アプリケーション用ビュー技術は JSP (JavaServer Pages) です。しかし JSP は『ビューとロジックとが切り分けられず、実行しなければデザインの確認ができない』 というビュー技術として致命的な欠陥を持っています。しかしJ-StarはJQueryをベースにしたJS技術でこの問題を解決します。
また既存のビュー技術ではサーバ側で次遷移先を決定していましたが、J-Starでは次画面遷移先の決定もJ-Star自身がおこなうため、サーバーはサービス処理だけに注力することが出来ます。



特徴


1. ローカルモードでの動的画面が作成可能
ローカルモードとはサーバーへの接続をOFFにした状態（サーバーが無い状態）を指します。 今までの紙芝居HTMLでは出来ない、ユーザーの入力に反応する画面が作成できます。 これにより画面の変更はテスト前まで可能になり、お客様と画面を考える時間を増やすことが出来ます。 また、設計フェーズではサービスI/Fの決定だけに注力することが出来ます。 システムが完成する前に挙動が目に見えるのでGDやリモートメンバの理解につながり品質向上が期待できます。

2.強力なデバッガ
各画面で「SHIFT + CTRL + ALT + SPACE」を同時に押下すると、ページ下部にデバッガ画面が現れます。 フォームなどの情報（画面定義書）、ログ、現在のHTMLが出力されます。 画面定義書は、選択された項目がハイライトされ今までの画面定義書では実現できなかった紐付が可能です。
※当機能はアドオンで提供しています

3.国際化
ブラウザで使用している言語に応じた画面が表示できます。デフォルトの言語のみ使用する場合HTMLにそのまま記述できるためメッセージファイルの作成は必要ありません。

4.豊富な便利機能
入力チェックやフォーマット処理が豊富に提供されています。 また、jQueryベースのため、J-Star以外のツールとも簡単に連携できます。
