validate(options)

フォームのテキストフィールドに入力チェックを設定します。
実際に入力チェックが実行されるのは、テキストフィールドのchangeイベントが発生した時と、このフォームが送信される直前に実行されます。
もし同一テキストフィールドにフォーマットが設定されている場合、 入力された値をアンフォーマットしたデータに対して入力チェックが実施されます。

またこのvalidate関数はエラー時メッセージ設定のvalidate_messageと共に使われるのが一般的です。


このvalidate関数の引数（options）はオブジェクトです。指定できる詳細は以下の通りです。

require / boolean
trueの場合必須チェックを行います。
ch / String
入力可能な文字種を指定します。指定可能な種は次の通りです。
これらは複数指定が可能です。"daA"とした場合、半角英数字を許可します
  • "d":半角数字
  • "D":半角数字以外
  • "a":半角英小文字
  • "A":半角英大文字
  • "w":半角英数字又は下線
  • "W":"半角英数字又は下線"以外
  • " ":半角空白
  • "$":記号（具体的には"!","(","]"などの文字コード33～47,58～64,91～96,123～126）
size / [int,int]
入力可能な文字長を指定します。
[1,2]の場合1文字以上2文字以下、[5,5]の場合5文字指定となります。
size / int
★現在バグ発生中★最低文字長チェックになっている★
入力可能な最大文字長を指定します。
size:2の場合、2文字以下の入力が可能です。
mask / patern
指定された正規表現オブジェクトによってパターンマッチングを行います。
例えば"mask:/^[0-9]a[0-9]$/"の場合、"0a7"はvalidateを通りますが"aaa","0aa"はエラーとなります。
range / [int,int]
数値で入力された値の存在範囲をチェックします。数値以外の文字が入っていた場合エラーとなります。例えば[5,10]の場合、5以上10以下の値が入力可能です。
custom / function
独自に実行する関数を定義します。関数の引数は入力値です。
戻り値

var isTOM = function(data){
  return data != "TOM";
};

$("[name=no]").validate({require:true, ch: "daA"})
  .valid_message("valid.req.def",[$("#no_label").html(),"valid.char.aAn"]);

  .valid_message("名前は半角英字か半角空白のみ許可。TOMは不可。");
