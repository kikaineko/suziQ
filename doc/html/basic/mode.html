J-Star

3つのモード

J-Starは動作モードとして以下の3つのモードを持っています。
これらのモードは実行できることが異なりますので、注意が必要です。

Local mode
Server mode
Production mode

Local mode
概要
ローカルモードは、サーバ無しにHTMLを開いたモードです。
つまり、HTMLファイルをダブルクリックするなどして立ち上げた状態になります。
これは、サーバ無しに画面の正確な挙動を確認するために利用されるモードです。

サーバのアクセスを代替するmockを自由に書き換えることによって まるでサーバが存在するかのような挙動をとらせることができます。
（ローカルモードとサーバモードの違いは、サーバからデータが貰えるかどうかだけですので画面内の挙動は完全に同一です）
ただし、ローカルモードにおいてもmock関数が定義されていない場合はサーバアクセスを試みられます。

このモードはSuziQによって自動で判別されます。
現在のHTMLのプロトコルが「file:」になっていると、ローカルモードだと判定されます。

他のモードとの違い
他のモードとローカルモードの最大の違いは、サーバにアクセスするかしないかですが、 他にもいくつかの違いがあります。

まず、ローカルモードでは国際化機能は動きません。 多くの場合OSの言語が取得されるため、ブラウザの言語を変更しても、変更は起きません。
これはJ-Starにおけるロケ－ルの取得方法がサーバへの問い合わせ結果になっているためです。

現在のロケールは「$$.locale();」の返り値で確認できます。


画面遷移の機能である「$$.ajax.to」関数が動作しない場合があります（サーバモード・Productionモードでは問題なく動作します）。
これはブラウザ毎に差が現れます。現在動作確認が取れているのは以下のブラウザです。

firefox 3 or 4
Safari 5
Opera 11
IE 7

逆に動作しないことが確認されているブラウザは以下です。
chrome
IE 6

繰り返しになりますが、これはローカルモード特有であり、サーバモード・Productionモードでは問題なく動作します。
Server mode
概要
サーバモードは、実際にサーバにアクセスを起こすモードです。 ある意味もっとも普通のモードといえます。

サーバモードはmock関数以外の全ての機能が使用できます。

このモードはJ-Starが自動で判定します。 つまり、ローカルモードでもProductionモードのどちらでも無ければ、サーバモードであると判定されます。

Production mode
概要
Productionモードは、サーバモードに制限を加えた本番用のモードです。

以下の機能が使用不可能になります。
デバッガー
ログ関数

ログ関数（$$.log）は、HTMLのJS内で呼ばれていても実際には何も実行されません。


Productionモード指定の2つの方法
1．このモードは自動では判定されません。ユーザが明示的に指定する必要があります。
以下のコードをsetting.jsに記述してください。

jQuery.sq.production=true;

2．J-Starではデバッガー機能を削除したjsファイルも配布しています。
こちらのファイルに変更するだけで本番モードと同一になります。
